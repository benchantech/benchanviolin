BENCHANVIOLIN JSON -> NEON POSTGRES -> SEARCH PAGE MVP PLAN
Scope: Build and test one complete deterministic retrieval loop on BenChanViolin.com.
Status: BenChanViolin.com is already Next.js. Do not redesign or migrate the rest of the website in this phase.

======================================================================
1. MVP OBJECTIVE
======================================================================

Prove this end-to-end flow with the Tan Dun YouTube Symphony Audition Walkthrough first:

Local Whisper JSON transcript files
  -> timestamp-accurate candidate evidence
  -> manually reviewed Tan Dun-style segment CSV
  -> Neon Postgres import
  -> deterministic tag / alias search
  -> clickable canonical tag
  -> reviewed segment card
  -> exact-start YouTube embed
  -> source-video link

The MVP succeeds when a visitor can type:

"bow bounce"

and the site deterministically surfaces:

Ricochet
  matched alias: "bow bounce"

Then, after clicking Ricochet, show a reviewed clip such as:

Bow-Drop / Ricochet — Flat Hair and Tension Diagnosis
Tan Dun YouTube Symphony Audition Walkthrough
5:43–6:00
Watch segment

The embedded player must begin at the stored exact start second.

No LLM, embeddings, user accounts, personalized routing, or bulk archive processing in this phase.

======================================================================
2. NON-GOALS FOR THIS MVP
======================================================================

Do not build:

- YY Method™ pages or routing.
- Violin Stand Partner integration.
- User profiles.
- Practice memory.
- AI recommendations.
- Full archive ingestion.
- Automated transcript interpretation.
- Multi-tag AND/OR filtering unless the one-tag loop is fully working.
- Analytics beyond optional simple no-result logging.
- A custom video host.
- End-time stopping logic in the YouTube embed.

The goal is one trustworthy public search loop, not a finished platform.

======================================================================
4. JSON -> REVIEWED SEGMENT CSV WORKFLOW
======================================================================

A. JSON IS THE TIMESTAMP AUTHORITY

Use Whisper JSON, not plain TXT, to determine segment timing.

Each JSON utterance contains:
- timestamps.from
- timestamps.to
- offsets.from
- offsets.to
- text

Critical rule:

Transcript times are relative to the chunk file.

Original-video time equals:

absolute_start_seconds = chunk_start_seconds + JSON_relative_start_seconds
absolute_end_seconds = chunk_start_seconds + JSON_relative_end_seconds

Example:

File:
0f6n1HMHI6g__chunk_0003__000600-000801.json

JSON utterance:
00:00:20,820 to 00:00:29,660

Absolute original-video range:
620.820 seconds to 629.660 seconds

B. EXTRACT EVIDENCE ROWS

Run the JSON extraction script across transcript folders.

Expected output:
- per-video utterance CSV
- one combined utterance CSV
- one review template CSV
- a manifest

The raw output is evidence only.

Do not import individual utterances into the public segment table.

C. CREATE REVIEWED SEGMENTS

A reviewed segment may combine multiple adjacent utterances.

For each candidate:

1. Read the JSON evidence.
2. Calculate exact original-video seconds.
3. Watch the original YouTube video at that range.
4. Adjust segment boundaries to avoid mid-sentence starts/ends.
5. Confirm the physical and visual explanation matches the transcript.
6. Write one concise learner-facing segment record.
7. Add only canonical tags.
8. Add a concrete "Use when" condition.
9. Add a short teaching summary.
10. Mark the row manual_reviewed only after actual review.

- initial tag groups
- initial tags
- aliases
- one video record

- segment tag assignments

Recommended indexing:

- GIN trigram index on tags normalized label.
- GIN trigram index on aliases normalized alias.
- btree index on segment_tags(tag_id, segment_id).
- btree index on segments(video_id, start_seconds).
- partial public index for:
  is_public = true
  and review_status = 'manual_reviewed'

Do not use a generic full-text transcript index for this MVP.
Search tags and aliases first.
The segment content should only appear after the user chooses a validated canonical concept.

======================================================================
7. DETERMINISTIC TAG SEARCH RULES
======================================================================

The tag typeahead should not search videos directly.

It should search canonical tags and aliases in this order:

1. Exact canonical label match.
2. Exact approved alias match.
3. Canonical prefix match.
4. Alias prefix match.
5. Trigram typo/fuzzy match.
6. Clip count only as a small tie-breaker.

Examples:

Query:
bow bounce

Expected top result:
Ricochet
Matched alias: bow bounce

Query:
tense bow hand

Expected top result:
Bow relaxation
Matched alias: tense bow hand

Query:
snap pizz

Expected top result:
Bartók pizzicato
Matched alias: snap pizzicato

The UI should expose why the tag was returned.
That transparency is a feature.

======================================================================
10. DATA IMPORT
======================================================================

A. FIRST IMPORT

Input:
Tan Dun reviewed canonical CSV.

Steps:

1. Validate the CSV locally.
2. Seed tag groups.
3. Seed canonical tags.
4. Seed aliases.
5. Upsert parent video.
6. Upsert segments using:
   video_id + start_seconds + end_seconds
7. Resolve tag_slugs to tag IDs.
8. Insert or update segment_tags.
9. Mark only manual_reviewed records public.
10. Run validation after import.

B. REQUIRED VALIDATION

Before a segment can become public:

- video_id is valid.
- start_seconds >= 0.
- end_seconds > start_seconds.
- start_seconds is within known video duration.
- segment key is unique.
- parent video exists.
- at least one tag slug resolves.
- review_status = manual_reviewed.
- is_public = true only when manually reviewed.
- title, use_when, and teaching_summary are non-empty.
- generated start URL resolves from stored video ID and start seconds.

C. IMPORT SCRIPT OUTPUT

The import script should print:

videos upserted
segments inserted
segments updated
tag assignments inserted
unknown tag slugs
invalid rows
duplicate segment keys
public segment count

Fail the import if unknown tag slugs or invalid public rows exist.

======================================================================
12. MVP DEFINITION OF DONE
======================================================================

The MVP is done when all of the following are true:

1. Existing Next.js BenChanViolin.com contains a public /library page.
2. Neon stores the Tan Dun source video, tags, aliases, and 17 reviewed segments.
3. A visitor can type a known learner phrase and see a matching canonical tag.
4. A visitor can click the canonical tag and view only manual-reviewed clips.
5. Each clip has a title, exact timestamp, tags, use condition, teaching summary, and source link.
6. Clicking Watch segment embeds YouTube at the exact stored start second.
7. The system does not require an LLM.
8. One import script can re-run without creating duplicate video or segment records.
9. The site can add a second video later by following the same JSON -> CSV -> import workflow.
10. The implementation is small enough that failure points are inspectable: JSON, CSV, database row, tag match, endpoint, card, embed.
